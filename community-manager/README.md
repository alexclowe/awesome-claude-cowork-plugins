# Community Manager Plugin for Claude

Moderate communities, onboard members, surface sentiment trends, and run event workflows.

Built by [The AI Career Lab](https://theaicareerlab.com/professions/community-manager) — AI tools, guides, and weekly digests designed specifically for community managers.

## Commands

| Command | Description |
|---------|-------------|
| `/moderate-community` | Generate moderation playbooks for spam/toxicity/harassment with platform-specific actions and escalation rules |
| `/onboard-member` | Draft a 7-day onboarding sequence with welcome messages, FAQ replies, role assignments, and first-interaction prompts |
| `/analyze-sentiment` | Aggregate community mood from a sample of messages, flag churn signals, and recommend engagement tactics |
| `/run-event-workflow` | Manage event registrations, reminder cadence, post-event follow-up, and participation reporting |

## Skills

This plugin includes skills that activate automatically when you're working on community management tasks:

- **Community Health Monitoring** — Track member engagement, retention, churn predictors; flag disengagement patterns early
- **Moderation Audit Trail** — Maintain compliance logs for corporate/regulated communities (financial services, healthcare); document escalation chains

## Usage examples

```
/moderate-community
Discord server, 4,200 members, fintech vertical. Recurring problem: crypto-shill
spam in #general (3-5 incidents/day), occasional harassment in DMs reported to
mods. Need a tiered response playbook with auto-mod rules, mod escalation, and
ban thresholds.
```

```
/onboard-member
Slack community for indie SaaS founders. New members frequently lurk and never
post. Goal: first message within 7 days, second within 14. Include welcome DM,
intro thread prompt, role-self-select, and a Day-3 nudge if no activity.
```

```
/analyze-sentiment
Sample of 200 messages from past 14 days in a customer Discord (B2B dev tool).
Recent product launch had bugs. Surface mood shift, flag potential churn voices,
and recommend three engagement tactics for this week.
```

```
/run-event-workflow
Virtual AMA with our founder, Thursday 2pm ET, expecting 300 RSVPs from a 6,000-
member community. Build the registration flow, three reminder messages (T-7d,
T-1d, T-1h), live-event runbook, and a 48-hour post-event follow-up sequence.
```

## Disclaimer

This plugin generates moderation drafts, onboarding copy, and engagement recommendations for community manager review. It does not replace human judgment on harassment, safety, or legal escalation decisions. For regulated communities (financial services, healthcare, education), verify outputs against your platform's compliance policies and applicable laws before publishing or acting.
