# Healthcare Compliance Officer Plugin for Claude

Validate AI medical devices, manage PCCPs, monitor adverse events, and stay aligned with FDA QSR.

Built by [The AI Career Lab](https://theaicareerlab.com/professions/healthcare-compliance-officer) — AI tools, guides, and weekly digests designed specifically for healthcare compliance officers.

## Commands

| Command | Description |
|---------|-------------|
| `/validate-ai-device` | Assess clinical validation for an AI-enabled medical device against QSR (21 CFR 820) and generate a documentation gap report |
| `/manage-pccp` | Draft or audit a Predetermined Change Control Plan and check whether a proposed modification stays in scope |
| `/monitor-adverse-events` | Ingest adverse event descriptions, assess causation, decide if FDA MedWatch reporting is required, and draft the report |
| `/assess-data-quality` | Evaluate training-data diversity for the intended-use population and flag bias / robustness gaps |

## Skills

This plugin includes skills that activate automatically when you're working on healthcare AI compliance tasks:

- **FDA Classification Advisor** — Determine the appropriate premarket pathway (510(k), De Novo, PMA) based on intended use and predicate landscape
- **Clinical Evidence Mapper** — Map study data to FDA Substantial Equivalence and clinical performance criteria

## Usage examples

```
/validate-ai-device
AI-enabled CT triage tool flags suspected intracranial hemorrhage and reorders
the radiologist worklist. Validation: retrospective study on 4,200 scans from
3 US sites. Sensitivity 94%, specificity 89%. Map to QSR (21 CFR 820), FDA
GMLP, and Jan 2026 post-market guidance. Identify documentation gaps before
510(k) submission.
```

```
/manage-pccp
Existing PCCP for our diabetic-retinopathy AI screening device. Proposed
modification: extend training data to include images from a new fundus camera
model and update the threshold for "more than mild DR" by 2 percentage points.
Confirm whether this stays inside the PCCP envelope or requires a new 510(k).
```

```
/monitor-adverse-events
Three reports in the last 30 days for our AI-enabled ECG interpretation device:
(1) missed STEMI in a 46yo, no patient harm, MD overread caught it; (2) false
positive AFib alert leading to unnecessary anticoagulation start, patient had
GI bleed; (3) device-down for 4 hours during a software push. Assess causation,
MedWatch reportability, and draft any required reports.
```

```
/assess-data-quality
Training data for our AI sepsis prediction model: 180,000 ICU encounters from
2 academic medical centers in the Northeast US, 2018–2024. Intended use
population: adult ICU patients in any US hospital. Evaluate diversity gaps
against the intended-use population and flag bias / robustness exposure.
```

## Disclaimer

All clinical and regulatory content generated by this plugin is for informational and drafting purposes only. It does not constitute medical, legal, or regulatory advice. The healthcare compliance officer is responsible for verifying all FDA citations and clinical claims against current authoritative sources, and consulting regulatory counsel and clinical leadership before any submission to FDA or use in patient care.
